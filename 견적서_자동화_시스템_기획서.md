# 정보통신사업부 견적서 자동화 시스템 기획서

> **프로젝트:** 정보통신설비 유지보수관리 견적서 자동화  
> **작성일:** 2026년 2월 27일  
> **목표:** AI Studio 기반 입력 UI → Google Sheets 자동 매핑 → PDF 견적서 발행

---

## 1️⃣ 프로젝트 개요

### 배경 및 목적
현재 견적서 작성은 Excel 파일(`260123_견적서양식_영업본부.xlsx`)에서 담당자가 대상처명을 직접 입력하면 각 시트의 수식이 연동되어 계산되는 방식으로 운영됩니다.

이 프로세스에는 다음과 같은 병목이 존재합니다:
- 엑셀 파일 공유 및 버전 관리의 어려움
- 시트별 PDF 수동 변환 작업
- 고객별 데이터 추적 불가
- 실수 발생 시 원인 파악 어려움

**이 프로젝트는 이를 해결하기 위해 아래 3단계 자동화 파이프라인을 구축합니다:**

```
[STEP 1] AI Studio (입력 UI)
      ↓ 데이터 전송 (API)
[STEP 2] Google Sheets (계산 및 매핑)
      ↓ PDF 변환
[STEP 3] 완성된 견적서 PDF 다운로드
```

#### 화면 C: 실시간 계산 결과 미리보기

입력값을 바탕으로 AI Studio 내에서 다음 값들을 **실시간 계산하여 표시**합니다.

| 표시 항목 | 계산 방법 |
|-----------|-----------|
| 건물 등급 (초급/고급/특급) | 연면적 → 견적 조건표 매핑 |
| 조정계수 | 연면적 → 조정계수 시트 매핑 |
| 성능점검 투입인원 | `Σ (수량 × 기준인원 × 조정계수)` → ROUNDDOWN |
| 성능점검 금액 | 견적 조건표 I열 (연면적 기준) |
| 유지점검 금액 | 견적 조건표 H열 (연면적 기준) |
| 선임 위탁 금액 | 연면적 기준 등급 판단 × 12개월 × 월단가 |
| **연간 총 견적금액** | 성능 + 유지 + 선임 합계 |
| **월 납부금액** | 연간 총액 ÷ 12 |

### 4-3. Apps Script Web App 구조

```javascript
// Code.gs (Google Apps Script)

const SPREADSHEET_ID = "YOUR_SHEET_ID_HERE";

function doPost(e) {
  const data = JSON.parse(e.postData.contents);
  const action = data.action;

  if (action === "writeQuotation") {
    return writeQuotationData(data.payload);
  } else if (action === "generatePDF") {
    return generatePDF(data.sheetNames);
  }
}

function writeQuotationData(payload) {
  const ss = SpreadsheetApp.openById(SPREADSHEET_ID);

  // 시트 1: 1. 견적서 - 기본 정보 쓰기
  const sheet1 = ss.getSheetByName("1. 견적서");
  sheet1.getRange("E8").setValue(new Date());        // 견적일
  sheet1.getRange("E9").setValue(payload.customerName); // 고객명
  // ... (나머지 셀 매핑)

  // 시트 6: 4. 성능점검 수량내역 - 설비 수량 쓰기
  const sheet6 = ss.getSheetByName("4. 성능점검 수량내역");
  const equipmentData = payload.equipmentList;
  equipmentData.forEach((item, i) => {
    sheet6.getRange(5 + i, 3).setValue(item.count); // C열에 수량 기입
  });

  // 시트 수식 재계산 대기
  SpreadsheetApp.flush();

  return ContentService
    .createTextOutput(JSON.stringify({ status: "success" }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

## 5️⃣ STEP 3: PDF 생성 및 다운로드

### 5-1. PDF 생성 방식

Google Sheets의 각 시트를 개별 PDF로 변환하거나, 1~6번 시트를 하나의 PDF로 묶어 출력합니다.

```javascript
function generatePDF(sheetNames) {
  const ss = SpreadsheetApp.openById(SPREADSHEET_ID);
  const folder = DriveApp.getFolderById("YOUR_FOLDER_ID");

  const customerName = ss.getSheetByName("1. 견적서")
                         .getRange("E9").getValue();
  const date = Utilities.formatDate(new Date(), "Asia/Seoul", "yyyyMMdd");
  const fileName = `${customerName}_견적서_${date}.pdf`;

  // 시트별 PDF 변환 URL 구성
  const exportUrl = `https://docs.google.com/spreadsheets/d/${SPREADSHEET_ID}/export` +
    `?format=pdf` +
    `&portrait=true` +
    `&fitw=true` +
    `&sheetnames=false` +
    `&printtitle=false` +
    `&pagenumbers=false` +
    `&gridlines=false` +
    `&fzr=false` +
    `&gid=SHEET_GID_HERE`; // 특정 시트 GID 지정

  // OAuth 토큰으로 PDF 바이트 가져오기
  const token = ScriptApp.getOAuthToken();
  const response = UrlFetchApp.fetch(exportUrl, {
    headers: { Authorization: `Bearer ${token}` }
  });

  // Google Drive에 저장
  const pdfFile = folder.createFile(response.getBlob().setName(fileName));

  return ContentService
    .createTextOutput(JSON.stringify({
      status: "success",
      pdfUrl: pdfFile.getDownloadUrl(),
      fileName: fileName
    }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

## 9️⃣ 주요 고려사항 및 리스크

> [!IMPORTANT]
> **Google Sheets 레이아웃 일치:** PDF 출력 결과가 원본 엑셀과 동일해야 합니다. Google Sheets는 셀 병합, 텍스트 방향, 폰트 등의 재현에 한계가 있을 수 있으므로, PDF 출력 전 레이아웃 검토가 필수입니다.

> [!WARNING]
> **CORS 이슈:** AI Studio에서 Apps Script Web App을 직접 `fetch()` 호출 시 CORS 문제가 발생할 수 있습니다. `window.open()` 또는 Proxy 서버 방식으로 우회가 필요할 수 있습니다.

> [!NOTE]
> **대상처 목록 동기화:** 현재 엑셀의 대상처 목록(1,100건+)은 Google Sheets로 최초 1회 이전 후, 신규 추가 시 Google Sheets에서 직접 관리합니다.

> [!TIP]
> **수식 유지 전략:** AI Studio는 `C열(수량)`과 기본 정보 셀만 씁니다. 금액 계산 수식(`H열`, `T열` 등)은 Google Sheets가 처리하므로 계산 오류 리스크가 최소화됩니다.
