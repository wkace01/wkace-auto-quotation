## 1. 프로젝트 개요

### 1-1. 시스템 목적

우경정보통신 정보통신사업부의 **소방설비 유지관리 서비스 견적서를 자동으로 생성**하는 웹 애플리케이션입니다.
기존에 수작업으로 처리하던 아래 3가지 업무를 하나의 시스템으로 통합합니다.

| 기존 방식 | 자동화 후 |
|----------|----------|
| 엑셀 수기 입력 | 주소 검색 → 건물 정보 자동 입력 |
| 계산식 직접 작성 | 연면적 기반 금액 자동 산출 |
| 견적서 PDF 수동 저장 | 버튼 한 번으로 PDF 생성 + DB 저장 |

### 1-2. 주요 기능 요약

- **주소 검색 → 건물 정보 자동 조회**: 카카오 주소 검색 후 공공데이터포털 건축물대장 API로 연면적·용도·사용승인일 자동 입력
- **견적 자동 계산**: 연면적 기준 등급 판별, 조건표 기반 성능점검·유지점검·위탁선임 금액 자동 산출, 배수·할인율 실시간 반영
- **PDF 견적서 생성**: Excel 템플릿에 데이터 주입 후 LibreOffice로 변환, 7페이지 통합 PDF 다운로드
- **Airtable DB 자동 저장**: 고객 정보 및 견적 기록을 Airtable에 자동 저장, PDF 첨부파일까지 업로드

### 1-3. 접속 방법

| 환경 | URL | 비고 |
|------|-----|------|
| **로컬 개발** | `http://localhost:3001` | `npm start` 실행 후 접속 |
| **배포 (Railway)** | Railway 대시보드에서 확인 | `feature/task3-airtable-fixes` 브랜치 연동 |

### 1-4. 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | HTML5, CSS3, Vanilla JavaScript (ES6+) |
| 백엔드 | Node.js + Express.js (포트 3001) |
| PDF 생성 | xlsx-populate (Excel 작성) + LibreOffice CLI (PDF 변환) |
| 데이터베이스 | Airtable (고객 / 견적 테이블) |
| 주소 검색 | 카카오 우편번호 API |
| 건물 정보 | 공공데이터포털 건축물대장 API |
| 배포 | Railway.app (GitHub 자동 배포) |

## 2. 화면 구성 및 사용 흐름

시스템은 **3단계 마법사(Step) 구조**로 구성되어 있습니다.

```
[Step 1: 주소 검색]
      ↓ 주소 선택 시 자동 이동
[Step 2: 건물 정보 입력 + 견적 조건 설정]
      ↓ 견적 완성 후 "PDF 생성" 또는 "DB 저장"
[Step 3: 견적 결과 상세 확인]
```

---

### Step 1 — 주소 검색

**목적**: 대상 건물의 주소를 선택하는 시작 단계

- 화면에 **카카오 우편번호 검색 위젯**이 임베드되어 있음
- 건물명 또는 도로명/지번을 검색하면 목록에서 선택 가능
- 주소 선택 시 자동으로 Step 2로 이동하고, 선택된 주소를 기반으로 **건축물대장 자동 조회** 시작

---

### Step 2 — 건물 정보 입력 및 견적 조건 설정

가장 많은 입력이 이루어지는 핵심 단계입니다. 화면은 **좌측 입력 패널**과 **우측 실시간 요약 패널**로 나뉩니다.

#### 좌측 입력 패널 구성

**① 건물 정보**

| 항목 | 자동/수동 | 설명 |
|------|----------|------|
| 선택된 주소 | 자동 | Step 1에서 선택한 도로명 주소 (수정 불가) |
| 고객명 (건물명) | 자동+수동 | 건축물대장 조회 결과로 자동 입력, 수동 수정 가능 |
| 연면적 (㎡) | 자동+수동 | 건축물대장에서 자동 입력, 수동 수정 가능 |
| 등급 분류 | 자동 | 연면적 기준 초급/중급/고급/특급 자동 표시 |
| 주용도 | 자동+수동 | 건축물대장에서 자동 입력 |
| 사용승인일 | 자동+수동 | 건축물대장에서 자동 입력 |
| 건축물대장 조회 버튼 | 수동 | 수동으로 재조회 필요 시 클릭 |

**② 거래처 담당자 정보**

| 항목 | 설명 |
|------|------|
| 담당자명 | 고객사 담당자 이름 |
| 직함 | 관리소장, 시설팀장 등 |
| 연락처 | 사무실 전화번호 (자동 포맷: 02-XXXX-XXXX) |
| 휴대전화 | 휴대폰 번호 (자동 포맷: 010-XXXX-XXXX) |
| 이메일 | 이메일 주소 |

**③ 영업 정보**

| 항목 | 설명 |
|------|------|
| 영업담당자 | 드롭다운 선택 (12명 목록 + 직접입력) |
| 영업담당자 연락처 | 담당자 선택 시 자동 입력, 직접입력 시 수동 입력 |
| 견적일 | 기본값: 오늘 날짜, 수동 변경 가능 |

**④ 견적 조건 패널** ← 가장 중요한 영역

> 연면적을 입력하면 조건표 기준 금액이 자동으로 채워집니다.
> 이후 각 항목을 필요에 따라 수동으로 조정할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 성능점검 비용 | 연간 금액 / 점검 횟수 입력 / ON-OFF 토글 |
| 유지점검 비용 | 연간 금액 / 점검 횟수 입력 / ON-OFF 토글 |
| 위탁선임 비용 | 월 단가 + 연간 합계 / 횟수 / ON-OFF 토글 |
| 할인율 | ±10% 버튼으로 조정 (0~100%) |
| 견적 조정 배수 | 미적용/적용 토글 + ±0.1 버튼 (기본값 1.5배) |
| 부가세 | 별도/포함 토글 (10%) |
| 투입 인원 | 성능점검 / 유지점검 각 투입 인원 수 |
| 조건 초기화 버튼 | 수동 수정한 금액을 조건표 기본값으로 되돌림 |

#### 우측 실시간 요약 패널

| 표시 항목 | 설명 |
|----------|------|
| 건물 등급 | 연면적 기준 등급 (초급/중급/고급/특급) |
| 조정계수 | 면적 구간별 계수 (참고용) |
| 연간 총 견적금액 | 배수·할인율·부가세 모두 적용된 최종 금액 |
| 월 견적금액 | 연간 금액 ÷ 12 |
| 성능점검 투입인원 | 해당 등급 기준 인원 수 |
| 유지점검 투입인원 | 해당 등급 기준 인원 수 |
| **PDF 생성 버튼** | 클릭 시 Excel 데이터 주입 → PDF 다운로드 |
| **Airtable DB 저장 버튼** | 고객·견적 정보 DB 저장 (PDF 첨부 포함) |

### Step 3 — 견적 결과 상세 확인

## 3. 핵심 기능 상세

### 3-1. 건축물대장 자동 조회

주소 선택 후 자동으로 진행되는 흐름:

```
카카오 주소 검색 완료
       ↓
공공데이터포털 도로명주소 API 호출
(지번코드 추출: 시군구코드 + 법정동코드 + 본번 + 부번)
       ↓
공공데이터포털 건축물대장 API 호출
(BldRgstHubService - XML 응답)
       ↓
주건축물(메인 동) 우선 선택
       ↓
연면적(totArea), 주용도(mainPurpsCdNm), 사용승인일(useAprDay), 건물명(bldNm) 자동 입력
       ↓
연면적 기준 등급 자동 판별 + 견적 조건 자동 계산
```

> **주의**: 건축물대장 조회는 인터넷 연결 필수. 실패 시 연면적을 수동 입력하면 나머지 계산은 정상 작동합니다.

---

### 3-2. 견적 자동 계산 로직

#### 등급 판별 기준

연면적 입력 시 아래 기준으로 등급이 자동 결정됩니다.

| 연면적 범위 | 등급 |
|------------|------|
| 5,000 ㎡ 이상 ~ 15,000 ㎡ 미만 | 초급 |
| 15,000 ㎡ 이상 ~ 30,000 ㎡ 미만 | 중급 |
| 30,000 ㎡ 이상 ~ 60,000 ㎡ 미만 | 고급 |
| 60,000 ㎡ 이상 | 특급 |

#### 조건표 기반 금액 산출

등급 및 연면적 구간이 결정되면 **견적 조건표**에서 기준 금액을 조회합니다.

| 연면적 구간 | 등급 | 성능점검(연) | 유지점검(연) | 위탁선임(월) | 위탁선임(연) | 투입인원 |
|-----------|------|------------|------------|------------|------------|---------|
| 5,000~10,000 | 초급 | 1,080,000 | 360,000 | 80,000 | 960,000 | 4명 |
| 10,000~15,000 | 초급 | 1,335,000 | 405,000 | 80,000 | 960,000 | 4명 |
| 15,000~30,000 | 중급 | 990,000 | 450,000 | 130,000 | 1,560,000 | 6명 |
| 30,000~60,000 | 고급 | 1,770,000 | 630,000 | 150,000 | 1,800,000 | 8명 |
| 60,000 이상 | 특급 | 2,430,000 | 810,000 | 180,000 | 2,160,000 | 10명 |

#### 배수 · 할인율 적용 방식

견적 조정 배수와 할인율은 **동시에 하나의 조정 계수(adjFactor)**로 합산하여 적용됩니다.

```
조정계수(adjFactor) = 견적조정배수 × (1 - 할인율/100)

최종 표시 금액 = 조건표 기준 금액 × adjFactor
```

**예시**: 기준 금액 1,000,000원 / 배수 1.5 / 할인율 10%
```
adjFactor = 1.5 × (1 - 0.1) = 1.35
최종 금액 = 1,000,000 × 1.35 = 1,350,000원
```

> 견적 조건 패널의 각 항목 금액 입력란에 표시되는 숫자는 항상 **조정계수가 이미 반영된 최종 금액**입니다.
> 배수/할인율 변경 시 실시간으로 숫자가 바뀝니다.

#### 항목 ON / OFF 토글

각 서비스 항목은 개별적으로 활성화/비활성화할 수 있습니다.

| 토글 상태 | 동작 |
|---------|------|
| ON | 조건표 기준 금액 정상 적용 |
| OFF | 해당 항목 금액 = 0, 투입 인원 = 0 |

> **위탁선임만 OFF** 상태에서 성능점검 또는 유지점검이 ON이면 "10% 인상 권장" 힌트가 표시됩니다.

#### 인건비 산출 방식 (산출내역 탭 기준)

투입 인원과 등급별 노임단가를 기반으로 인건비를 역산합니다.

```
직접인건비 = 투입인원 × 등급별 노임단가
직접경비   = 직접인건비 × 10%
제경비     = 직접인건비 × 110%
기술료     = (직접인건비 + 제경비) × 20%
─────────────────────────────────────────
산출 합계  = 직접인건비 + 직접경비 + 제경비 + 기술료
조정금액   = 목표금액(견적조건표 금액) - 산출합계
최종합계   = 목표금액 (조정금액으로 차이 보정)
```

### 3-3. 견적 조건 수동 조정

자동 산출된 금액을 담당자가 직접 조정할 수 있는 방법은 3가지입니다.

#### ① ± 버튼 (단계 조정)

- **금액 항목**: ±50,000원 단위 조정
- **인원 항목**: ±1명 단위 조정
- 배수·할인율이 적용된 **표시 금액 기준으로 조정**됩니다

#### ② 직접 입력

- 금액 입력란을 클릭하여 직접 숫자 입력
- 쉼표(,) 포함 숫자도 자동 파싱
- 월 단가와 연간 합계는 **연동**: 하나를 바꾸면 나머지가 자동 계산

#### ③ 조건 초기화 버튼

- "조건 초기화" 클릭 시 수동 수정된 모든 금액을 조건표 기본값으로 되돌림
- 배수·할인율 설정은 유지됨

---

### 3-4. PDF 견적서 생성

"PDF 생성" 버튼 클릭 시 아래 프로세스가 자동으로 진행됩니다.

```
① 화면의 모든 입력값 수집
        ↓
② Excel 셀 매핑 데이터 생성
   (시트명 → 셀 주소 → 입력값 쌍)
        ↓
③ 서버(/generate-pdf)로 전송
        ↓
④ 서버: Excel 템플릿(ver2.xlsx)에 데이터 주입
        ↓
⑤ LibreOffice CLI로 PDF 변환 (headless)
        ↓
⑥ PDF 파일 브라우저 다운로드
        ↓
⑦ (백그라운드) Airtable 첨부파일 업로드
```

#### 생성되는 PDF 파일명 규칙

```
{견적고유ID}_견적서_{고객명}_{담당자명}_{직함}님.pdf

예) QUOTE-20260324-001_견적서_휴먼타워_홍길동_관리소장님.pdf
```

#### Excel 템플릿 시트 구성 (`정보통신사업부 견적서 양식_ver2.xlsx`)

| 시트명 | 역할 | 주요 데이터 입력 위치 |
|--------|------|---------------------|
| **표지** | 커버 페이지 | A10: 고객명 ("고객명 : XXX" 형식), A18: 견적일 |
| **1. 견적서** | 최종 견적서 본문 | T24~T26: 항목별 금액, T27: 소계, T28: 최종연간금액, T29: 월 납부액 |
| **2.1 성능점검 산출내역** | 성능점검 인건비 산출 | E6~E9: 등급별 투입인원, G6~G9: 노임단가, H10~H17: 인건비 계산 결과 |
| **2.2 유지점검 산출내역** | 유지점검 인건비 산출 | 성능점검과 동일 구조 |
| **2.3 선임 산출내역** | 위탁선임 산출 | 선임 인건비 + 연간 합계 |
| **4. 성능점검 수량내역** | 조정계수 | F4: 연면적 조정계수 |

> 이외 시트는 PDF 생성 시 자동으로 숨김 처리되어 최종 PDF에 포함되지 않습니다.

### 3-5. Airtable DB 저장

"DB 저장" 버튼 클릭 또는 PDF 생성 시 자동으로 아래 순서로 저장됩니다.

```
① 도로명 주소 + 건물명으로 기존 고객 검색
        ↓
② 고객 존재 시: 정보 업데이트 (PATCH)
   고객 없을 시: 신규 생성 (POST)
        ↓ (350ms 대기 — Airtable 레이트 리밋 방지)
③ 견적 기록 생성 (고객 ID 연결)
        ↓
④ (백그라운드) PDF 파일을 견적 레코드에 첨부
```

#### Airtable 테이블 구조

**고객 테이블** (`고객`)

| 필드 | 저장 내용 |
|------|----------|
| 건물명 | 고객명 (건물명) |
| 도로명 주소 | 카카오 검색 결과 도로명 |
| 지번 주소 | 지번 주소 |
| 우편번호 | 우편번호 |
| 지역 | 시/구 단위 추출값 (예: 수원, 서울) |
| 연면적 | 건물 연면적 (㎡) |
| 주용도 | 건물 주용도 |
| 사용승인일 | 건물 사용승인일 |
| 담당자 / 직함 / 전화번호 / 휴대전화 / 이메일 | 거래처 담당자 정보 |

**견적 테이블** (`견적`)

| 필드 | 저장 내용 |
|------|----------|
| 고객 (링크) | 고객 테이블 레코드 연결 |
| 견적 고유 ID | 고유 식별자 |
| 견적 금액 | 최종 연간 금액 |
| 견적서 발송일 | 견적일 |
| 서비스 유형 | 성능점검 / 유지점검 / 위탁선임 (선택된 항목만) |
| 영업담당자 | 담당 영업사원 이름 |
| 성능점검 횟수 / 유지점검 횟수 / 위탁선임 횟수 | 각 항목 횟수 |
| PDF 첨부 | 생성된 견적서 PDF 파일 |

## 5. 마스터 데이터 관리

모든 마스터 데이터는 `public/constants.js` 파일에 집중 관리됩니다.
데이터 변경 시 이 파일만 수정하면 전체 시스템에 반영됩니다.

### 5-1. 견적 조건표 (`QUOTATION_CONDITIONS`)

연면적 구간별 기준 금액 및 투입 인원 데이터

```
수정 방법: public/constants.js → QUOTATION_CONDITIONS 배열
```

| 필드명 | 설명 |

| `area` | 연면적 구간 시작값 (㎡) |

| `yearlyAppointment` | 위탁선임 연간 금액 (원) |

| `yearlyInspection` | 성능점검 연간 금액 (원) |
| `inspectionWorkers` | 성능점검 투입 인원 (명) |
| `maintenanceWorkers` | 유지점검 투입 인원 (명) |

## 6. 운영 환경

### 6-1. 로컬 실행 방법

**사전 준비물**
- Node.js 설치 (v18 이상 권장)
- LibreOffice 설치 (PDF 변환용)
- `.env` 파일에 Airtable API Key 설정

**실행 명령어**

```bash
# 의존성 설치 (최초 1회)
npm install

# 서버 실행 (포트 3001)
npm start

# 브라우저 접속
http://localhost:3001
```

### 6-2. 배포 환경 (Railway)

| 항목 | 내용 |
|------|------|
| 플랫폼 | Railway.app |
| 배포 방식 | GitHub 연동 자동 배포 |
| 연동 브랜치 | `feature/task3-airtable-fixes` (PR 머지 후 `main` 브랜치 연동 권장) |
| 빌드 시스템 | Nixpacks (LibreOffice 자동 설치) |
| 포트 | Railway에서 자동 할당 (`PORT` 환경변수 사용) |

### 6-3. 환경변수 목록

`.env` 파일 (로컬) 또는 Railway 환경변수 설정 화면에서 관리

| 변수명 | 필수 여부 | 설명 |
|--------|---------|------|
| `AIRTABLE_API_KEY` | **필수** | Airtable Personal Access Token |
| `PORT` | 선택 | 서버 포트 (기본값: 3001) |

> **보안 주의**: `.env` 파일은 절대 Git에 커밋하지 마십시오. `.gitignore`에 등록되어 있습니다.

### 6-4. LibreOffice 설치 요건

PDF 생성 기능은 LibreOffice CLI(헤드리스 모드)에 의존합니다.

| 환경 | 설치 방법 |
|------|----------|
| **Windows (로컬)** | [LibreOffice 공식 사이트](https://www.libreoffice.org) → Windows 버전 설치 (`C:\Program Files\LibreOffice`) |
| **Linux (배포)** | Railway/Render Nixpacks가 자동 설치 |
| **macOS (로컬)** | `brew install --cask libreoffice` |

> LibreOffice가 없으면 PDF 생성 버튼 클릭 시 서버 오류가 발생합니다.
> `/health` 엔드포인트(`http://localhost:3001/health`)로 서버 상태를 확인할 수 있습니다.
